## What is Taix?

Taix is a **lightweight, native GUI application** for managing multiple terminal sessions — think iTerm2 or Windows Terminal, but cross-platform and with deep AI integration built in from the ground up.

**Core philosophy:** The terminal is the most direct interface between a human and a machine. AI should integrate naturally into that interface, not live in a separate window.

**Target users:** DevOps engineers, backend developers, system administrators.

### Key Features

- **Multi-tab terminal** — open multiple shell sessions, switch between them without losing state
- **Keep-alive tabs** — switching tabs never destroys the xterm.js instance; long-running processes (`top`, `vim`, etc.) keep running
- **SSH session manager** — save SSH profiles (password / private key / SSH agent auth), reconnect in one click, animated connecting overlay
- **Split panes** — horizontal/vertical terminal splits with draggable dividers
- **Session persistence** — saved sessions and groups stored in SQLite, survive restarts
- **In-terminal search** — scrollback search via `⌘F`
- **Cross-platform PTY** — native PTY on macOS/Linux (POSIX), ConPTY on Windows
- **Persistent configuration** — appearance, shell, and AI settings saved to TOML on disk
- **Settings UI** — font, theme (dark/light/system), default shell, all editable in-app
- **Type-safe IPC** — Rust ↔ TypeScript bindings auto-generated by [tauri-specta](https://github.com/oscartbeaumont/tauri-specta)

### Planned Features (Phase 3 — AI integration)

- **AI assist** — natural language → command (`#` trigger), error analysis, command suggestions
- **Ollama support** — fully local AI, no API key required
- **AI Agent mode** — AI executes commands with per-action confirmation and full audit log

## Getting Started

### Prerequisites

| Tool | Version | Install |
|---|---|---|
| Rust | ≥ 1.80 | `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs \| sh` |
| Node.js | ≥ 20 | [nodejs.org](https://nodejs.org) |
| npm | ≥ 9 | bundled with Node.js |

**Linux only** — install WebKit and GTK system dependencies:
```bash
sudo apt-get install -y libwebkit2gtk-4.1-dev libssl-dev libgtk-3-dev \
  libayatana-appindicator3-dev librsvg2-dev
```

### Run in development

```bash
git clone https://github.com/rynvaro/taix.git
cd taix
npm install
npm run tauri dev
```

### Build for production

```bash
npm run tauri build
```

Outputs are placed in `src-tauri/target/release/bundle/`:
- **macOS** → `.dmg` + `.app.tar.gz`
- **Linux** → `.AppImage` + `.deb`
- **Windows** → `.exe` (NSIS installer) + portable `.zip`

> **macOS note:** Downloaded builds are not code-signed. If macOS shows "app is damaged and can't be opened", run:
> ```bash
> xattr -cr /Applications/Taix.app
> ```
> then open again. This clears the quarantine flag set by Gatekeeper.

## Development

### Available scripts

```bash
npm run tauri dev      # Start development server + native window
npm run tauri build    # Build production bundle
npm run type-check     # TypeScript type check (tsc --noEmit)
npm run lint           # ESLint
npm run lint:fix       # ESLint with auto-fix
```

### Rust tests

```bash
cd src-tauri
cargo test             # Run all unit + integration tests
```

### Regenerate TypeScript bindings

TypeScript bindings (`src/types/bindings.ts`) are auto-generated from Rust types on every debug build or test run:

```bash
cd src-tauri && cargo test bindings_contain_expected_symbols
```

## CI / CD

Every push and PR runs three parallel jobs on **Linux, macOS, and Windows**:
- Rust unit + integration tests (`cargo test --all`)
- TypeScript type-check + ESLint

Releases are triggered by pushing a `v*` tag and automatically produce installers for all three platforms via [tauri-action](https://github.com/tauri-apps/tauri-action).

## Keyboard Shortcuts

See the [Usage Guide → Keyboard Shortcuts](docs/usage.md#keyboard-shortcuts) for the full reference.

| Shortcut | Action |
|---|---|
| `⌘T` | New tab |
| `⌘W` | Close active pane / tab |
| `⌘1`–`⌘9` | Switch to Nth tab |
| `⌘D` / `⌘⇧D` | Split pane horizontal / vertical |
| `⌘F` | In-terminal search |
| `⌘[` / `⌘]` | Focus previous / next pane |

> On Linux / Windows replace `⌘` with `Ctrl`.
